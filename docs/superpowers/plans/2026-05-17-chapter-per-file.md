# One JSX File Per Chapter Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Split the 27 section JSX files into 232 per-chapter JSX files (one chapter = one default-export function = one file), preserving runtime behavior, content, and test coverage bit-for-bit.

**Architecture:** Each chapter lives at `src/chapters/<section-topic>/<chapter-kebab>.jsx`. Shared helpers move to `src/shared/<topic>.jsx`. `config.js` gains a `file` field pointing each chapter at its file. `learn-ai.jsx` swaps the per-section `sectionLoaders` dictionary for a `import.meta.glob('./chapters/**/*.jsx')` map and looks up by `file`. The 9510-line `sections.test.jsx` is split into 232 per-chapter test files mirroring the chapter folder layout. The search-index build script switches from section-source hashing to chapter-source hashing.

**Tech Stack:** React 18, Vite, Vitest, Prettier, ESLint, MiniSearch, @huggingface/transformers (ONNX). Node 22+. Project uses inline styles, dark theme, no external UI libs.

**Spec:** `docs/superpowers/specs/2026-05-17-chapter-per-file-design.md`

## Conventions used in every chapter file

```jsx
// src/chapters/<topic>/<chapter>.jsx
import { C } from "../../config.js";
import { Box, T, Reveal, SubBtn /*, Tag */ } from "../../components.jsx";
// import { Graph } from "../../shared/plot.jsx";       // if used
// import { SOFT, pill } from "../../shared/agent-styles.jsx";   // if used

export default function ChapterFunctionName(ctx) {
  // body copied verbatim from the corresponding section file's named export
}
```

**Rules:**
- Default export only. The function name is the old PascalCase identifier (e.g., `WhySoftmax`). Devtools and stack traces stay readable.
- No exports other than `default`. No top-level helpers. Any helper used by 2+ chapters is in `src/shared/`. Any helper used by exactly one chapter goes inside that chapter's file as a module-private `const`/`function`.
- Relative imports use `../../config.js`, `../../components.jsx`, `../../shared/<file>.jsx`. Two levels up because chapter files live two directories deep.
- Function bodies are copied verbatim from the section file. Do not refactor, rename, or "tidy" while moving. Whitespace, comments, JSX, and inline data preserved exactly.
